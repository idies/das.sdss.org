Checking Flat-Files Against the Data Model

TCL procs are provided to check the format of flat-files against the data model. Documentation for these procs may be found at:
  • imgCheck.tcl --- Procs to check the imaging DA and pipelines
  • mtCheck.tcl --- Procs to check the MT DA and pipelines
  • paramCheck.tcl --- Procs to check the imaging DA and pipelines
  • fitsCheck.tcl --- Procs to check the imaging DA and pipelines
  • planCheck.tcl --- Procs to check a pipeline plan file
    • As an example, first, one must setup the data model: 
    sdssdp1> setup dm
    Next, start up dervish and source the dm startup file: 
    sdssdp1> dervish
dervish> source $env(DM_DIR)/etc/dmStartup.tcl
    Now go to the directories containing the report/plan files and run the checks. For example, lets say we're checking reductions of run 211: 
    sdssdp1> cd /data/dp3.b/data/fixed/211
sdssdp1> imgCheck idReport-51115.par
sdssdp1> cd /data/dp3.b/data/211/bias
sdssdp1> biasCheck biPlan.par
sdssdp1> cd /data/dp3.b/data/211/ssc
sdssdp1> sscCheck scPlan-1.par
sdssdp1> cd /data/dp3.b/data/211/astrom
sdssdp1> astromCheck asPlan.par
sdssdp1> cd /data/dp3.b/data/211/photo
sdssdp1> psCheck psPlan-1.par
sdssdp1> framesCheck fpPlan-1.par
sdssdp1> cd /data/dp3.b/data/211/nfcalib
sdssdp1> fcalibCheck fcPlan-1.par
    The name of each file being checked is printed to the screens, along with any inconsistencies with the data model.
